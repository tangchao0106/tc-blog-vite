# 大模型如何"看懂"文字：从 One-Hot 到词嵌入

## 计算机的"文字盲"问题

计算机只认识数字。给它"苹果"两个字，它看见的是：

```text
"苹果" → [0xE8, 0x8B, 0xB9, 0xE6, 0x9E, 0x9C]（UTF-8 编码的字节序列）
```

这串十六进制数字对计算机来说没有任何"苹果是可以吃的红色水果"这层语义。**如何把离散的文字变成计算机能"理解"的数字表示，是 NLP 的第一个核心问题。**

## 第一代方案：One-Hot 编码

最直观的想法——给每个词分配一个唯一编号，用一个超长向量表示，只有对应位置是 1，其余全 0。

```text
词表: ["苹果", "香蕉", "汽车", "电脑", ...]  (共 100,000 个词)

"苹果" → [1, 0, 0, 0, ..., 0]  (100,000 维)
"香蕉" → [0, 1, 0, 0, ..., 0]  (100,000 维)
"汽车" → [0, 0, 1, 0, ..., 0]  (100,000 维)
```

### One-Hot 的三个致命缺陷

**缺陷一：维度灾难**
词表 10 万个词，每个词就需要 10 万维的向量。今天的大模型词表动辄 10 万+，One-Hot 的向量长度就是 10 万维——内存和计算都不可接受。

**缺陷二：语义完全丢失**

```text
cosine_similarity("苹果", "香蕉") = 0  ← 都是水果，但 One-Hot 认为毫无关系
cosine_similarity("苹果", "汽车") = 0  ← 完全不同类，但也等于 0
```

在 One-Hot 的世界里，**任意两个不同的词的距离都一样远**。"苹果"、"香蕉"、"汽车"三者互为垂直向量，没有任何"苹果和香蕉更接近"的信息。

**缺陷三：无法泛化**

模型看到"国王 - 男人 + 女人 = ?"，One-Hot 无法给出"女王"——它不知道"国王"和"男人"之间有什么语义关系。

## 第二代方案：Word2Vec 与稠密向量

2013 年，Google 的 Tomas Mikolov 团队提出了 Word2Vec，彻底改变了 NLP。

### 核心思想

**用一个低维稠密向量（Dense Vector）来表示每个词，向量的数值通过学习得到，能捕捉词的语义。**

```text
# One-Hot（稀疏，10万维）
"苹果" → [1, 0, 0, 0, ..., 0]

# Word2Vec 嵌入（稠密，300维）
"苹果" → [0.23, -0.51, 0.78, 0.12, ..., -0.33]  (300维，每个值都是有意义的小数)
```

### 两种训练方式

**CBOW（Continuous Bag of Words）**：用上下文预测中心词

```text
输入：["我", "吃", "一个", "?"]
        ↓ 模型预测
输出："苹果"  ← 大概率是食物类词汇
```

**Skip-gram**：用中心词预测上下文

```text
输入："苹果"
        ↓ 模型预测
输出：["吃", "水果", "红色", "甜", ...]  ← 学习"苹果"的共现词
```

### Word2Vec 的魔法：语义运算

因为向量的每个维度都编码了某种语义特征，向量之间可以进行有意义的运算：

```text
vec("国王") - vec("男人") + vec("女人") ≈ vec("女王")
vec("巴黎") - vec("法国") + vec("意大利") ≈ vec("罗马")
vec("跑步") - vec("跑") + vec("游泳") ≈ vec("游")
```

这证明了稠密向量确实捕捉到了词语间的**语义关系和类比关系**。

### 可视化理解

我们可以把 300 维向量降维到 2D 来看：

```text
        ┌─ 苹果
        ├─ 香蕉        ← 水果类聚集在一起
        ├─ 橘子
        │
        ├─ 汽车        ← 交通工具类聚集在另一区域
        ├─ 火车
        ├─ 飞机
        │
        ├─ 电脑        ← 电子设备类
        ├─ 手机
```

**语义相近的词，在向量空间中离得近。** 这是 One-Hot 完全做不到的。

## 现代方案：学习型嵌入层

今天的大语言模型不再单独使用 Word2Vec，而是**在模型的第一层内置一个 Embedding 层**，作为整个神经网络的一部分端到端训练。

```python
import torch
import torch.nn as nn

# 大模型的第一层：嵌入层
class EmbeddingLayer(nn.Module):
    def __init__(self, vocab_size, embed_dim):
        super().__init__()
        # 词表 128,000 个 token × 4096 维嵌入
        self.embedding = nn.Embedding(vocab_size, embed_dim)

    def forward(self, token_ids):
        # token_ids: [batch, seq_len]  例如 [1, 100]
        # 输出:     [batch, seq_len, embed_dim]  例如 [1, 100, 4096]
        return self.embedding(token_ids)

# GPT-3 的嵌入层参数
vocab_size = 50257    # 词表大小
embed_dim  = 12288    # 嵌入维度
# 嵌入层参数量 = 50257 × 12288 ≈ 6.17 亿参数
```

### 嵌入维度的意义：为什么是 768 或 4096

你可能注意到了，不同模型的嵌入维度不同：

| 模型 | 嵌入维度 | 说明 |
|------|:------:|------|
| BERT-base | 768 | 早期标准 |
| BERT-large | 1024 | |
| GPT-3 | 12288 | 大模型倾向高维度 |
| LLaMA 2 70B | 8192 | |
| DeepSeek V3 | 7168 | |

**维度越高，向量能编码的语义信息越丰富，但计算量也越大。**

```text
嵌入维度 = 2：
  "苹果" → [0.8, -0.3]
  "香蕉" → [0.7, -0.2]
  能表达的信息：只有 x 轴和 y 轴两个方向

嵌入维度 = 768：
  "苹果" → [0.2, -0.5, 0.7, ..., 0.1]  (768个维度)
  能表达的信息：768 个语义方向
  每个方向可能编码一种语义特征：
    - 维度 1～50： 食物相关度
    - 维度 51～100：大小/尺寸
    - 维度 101～200：情感倾向
    - 维度 201～300：抽象/具体程度
    ... 以此类推
```

实际上这些维度对应什么语义是**模型自己学出来的**，人类不一定能解释清楚每个维度的含义。这也是深度学习"黑盒"特性的一部分。

## Subword Embedding：Token 级别的嵌入

现代 LLM 的嵌入层做在 **Subword（子词）** 级别，而不是完整词语级别。

```text
"tokenization" → ["token", "ization"]
"unbelievable" → ["un", "believe", "able"]

每个子词都有独立的嵌入向量：
  emb("token")     → [0.12, -0.34, ..., 0.56]
  emb("ization")   → [0.78, -0.21, ..., 0.33]
  emb("un")        → [-0.45, 0.67, ..., -0.12]
```

这样做的好处：
- **覆盖未登录词**：没见过"ChatGPTization"也能通过子词组合理解
- **词表可控**：128K 个子词就够了，不需要为每个完整词预留位置
- **跨语言共享**：中英文共用同一个子词表，英文天然由字母组成子词

## 从前端视角理解嵌入

如果你做过前端开发，可以把嵌入理解为**将离散标识映射为 CSS 特性向量**：

```javascript
// 每个 UI 组件有一个"特征向量"
const buttonFeatures = {
  interactive: 0.9,   // 可交互程度
  textual: 0.3,       // 文字化程度
  container: 0.1,     // 容器化程度
  focusable: 0.8,     // 可聚焦程度
  // ... 更多维度
}

const divFeatures = {
  interactive: 0.1,
  textual: 0.2,
  container: 0.9,
  focusable: 0.1,
}

// button 和 input 更相似（都是交互元素）
// div 和 section 更相似（都是容器元素）
```

这就是嵌入的本质：**在连续空间中给每个离散对象找一个"合适的位置"，位置之间的距离反映语义相似度。**

## 总结

| 阶段 | 表示方式 | 维度 | 语义信息 |
|------|---------|:---:|:------:|
| One-Hot | 稀疏向量 | 词表大小 | ❌ 无 |
| Word2Vec | 稠密向量 | 300 | ✅ 基本语义 |
| 现代 LLM Embedding | 学习型稠密向量 | 768～12288 | ✅✅ 丰富语义 |

> 从 One-Hot 到词嵌入，是从"死记硬背编号"到"理解语义关系"的飞跃。
>
> 嵌入层是现代大语言模型理解文本的**第一块基石**。没有它，后面所有的 Transformer、注意力机制都无从谈起。
>
> 记住这个直觉：**嵌入就是给每个词找一个"高维空间中的坐标"，让语义相近的词挨在一起。**
