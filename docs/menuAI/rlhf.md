# RLHF：让大模型"学会做人"

你有没有过这种感觉——大模型肚子里有货，但说话方式让人血压升高？比如你问它怎么煎蛋，它给你写了个分子料理的学术论文；你让它帮忙写封邮件，它开头就是"尊敬的先生/女士"。这种感觉就像是：一个天才，但没被教育过怎么跟人相处。

RLHF（人类反馈强化学习）就是干这个的——它不是让模型更聪明，而是让模型**更像个人**。

## 预训练学的是什么：不是"对"，是"像"

先说说预训练阶段学的是什么。说白了就是：看到一个词，下一个词最可能是什么。这是通过海量文本喂出来的，比如"床前明月光"后面最可能是"疑是地上霜"。

但预训练的目标是"预测得准"，不是"回答得好"。你可以理解为：预训练把大模型变成了一个超级模仿秀选手——它模仿的是整个互联网的说话风格，里面有知乎长文、有论坛杠精、有中二少年、也有客服机器人。

结果就是：你问它问题，它可能像个辩论选手一样先来一堆背景铺垫；你让它写个请假条，它可能给你整出个法律合同的架势。

## SFT做了什么：照着"正确答案"抄

为了让模型学会"正常说话"，第一步通常是监督微调（SFT）。做法很简单：找一批高质量的问答对，让模型照着抄。

打个比方就是：预训练像是让一个人读完了整个图书馆，里面什么书都有；SFT就是找了几本"职场沟通指南"让他重点看一遍。

SFT的效果立竿见影——模型确实变得会回答问题了。但它的问题是：它只是在模仿"看起来对"的答案，并不真正理解"为什么这个回答比那个好"。就像学生背了标准答案，考试换了个问法就抓瞎。

## 奖励模型：给回答打个分

RLHF的核心创新在于引入了"奖励模型"这个环节。

怎么理解奖励模型？想象你在教一条狗新把戏。你不会说"坐下——等等不对，应该再往左偏一点"这么精确的指令，你只会说"好"或者"不对"。狗不需要懂语法，它只需要知道：刚才那个动作，让主人高兴了。

训练奖励模型的思路类似：

1. 先让人对各种回答打分（比如"这个回答好/一般/差"）
2. 用这些人工标注的数据训练一个"奖励模型"
3. 这个模型学会了预测：给定一个问题和回答，它会打几分

有个关键细节：标注者通常比较的是**两个回答哪个更好**，而不是单独给每个回答打分。这种"对比"比"绝对评分"一致性更高——就像选秀节目里，评委说"A比B好"比单独给每个人打分更容易达成共识。

## PPO：在大海里学游泳

有了奖励模型，下一步就是让大模型根据这个奖励来调整自己的输出。这里面用的核心算法叫 PPO（近端策略优化），名字听起来唬人，背后的逻辑却很直观。

想象你在玩一个游戏，规则很复杂，你不知道每一步具体加多少分。你只能玩很多局，然后慢慢摸索：哪些操作经常拿高分，哪些操作经常被惩罚。玩得越多，你的"手感"就越准。

PPO 就是这个"摸索"的过程。只不过它聪明在：它不会一次改变太大——如果你刚找到一个不错的策略，就别急着冒险大幅调整，先稳一稳。这就是"近端"的意思：每次只迈一小步。

还有个技术细节叫 KL 散度约束。KL 散度可以理解为"两个分布有多不像"。加这个约束的原因是：纯粹的奖励最大化会导致模型走偏——比如一个取巧的方式是模型学会"讨好"奖励模型本身，而不是真正产生好回答。KL 约束限制了新策略偏离 SFT 模型的程度，防止模型"作弊"。

用一个生活比喻：KL 约束就像是你在学游泳，教练说"你可以尝试新姿势，但别一下子把自己扭成麻花"。这样既能进步，又不至于离谱。

## DPO：更简单的新路线

PPO 效果好，但训练流程太复杂——要同时跑奖励模型、参考模型、策略模型，还要小心各种调参。于是 2023 年有人提出了 DPO（直接偏好优化），把流程大大简化了。

DPO 的核心思路是：**不需要单独训练奖励模型，直接用偏好数据更新模型**。

怎么做到的？DPO 做了一个数学变换，把"奖励最大化 + KL 约束"的问题转化成了一个等价的"分类"问题。你可以理解为：原来你要训练两个模型（奖励模型 + 策略模型），现在只需要一个模型，但这个模型的"内部"同时扮演两个角色。

实际效果上，DPO 在很多任务上接近 PPO 的水平，但训练更简单、更稳定。当然它也有自己的局限性——比如对数据质量的要求更高，偏好数据的标注要更一致。

## 为什么 RLHF 这么重要

说了这么多技术细节，你可能想问：RLHF 到底解决了什么问题？

简单说就是三件事：

1. **对齐（Alignment）**：让模型输出的风格、语气、逻辑符合人类预期。不是"能说什么"而是"该怎么说"。
2. **安全（Safety）**：减少有害内容、虚假信息的生成。这不是靠规则过滤，而是让模型真正"理解"什么不该说。
3. **有用（Helpfulness）**：让模型真正帮上忙，而不是给出一堆看似正确但没用的废话。

ChatGPT 当年之所以惊艳，很多人觉得是 GPT-3 之后最大的突破，其实核心就在 RLHF。它让大模型从"能说话"变成了"会说话"。

## 写在最后

理解 RLHF 对前端开发者有什么用？一个很直接的价值是：当你用 API 调用大模型时，理解它的输出特性——比如为什么会"hallucinate"（幻觉）、为什么有时候回复很正式——能帮你更好地设计 Prompt 工程和结果后处理。

RLHF 让模型"更像人"，但它依然是工程和数学的产物。理解它的原理，能帮你用好这个工具，而不是被它的局限绊倒。
