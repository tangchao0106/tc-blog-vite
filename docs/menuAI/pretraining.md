# 预训练：大模型的"通识教育"

## 一句话解释预训练

**预训练（Pre-training）**是大模型学会"说话"的第一步。它让模型在海量文本中浸泡，积累对语言规律、世界知识甚至推理能力的隐性理解——这个阶段不教它具体任务，只让它"读书"。

如果把大模型比作一个孩子，预训练就是从出生到小学毕业的阶段：大量输入信息，吸收常识，建立对世界的基础认知。后续的微调（Fine-tuning）才是针对具体技能的专业培训。

## 为什么预训练最重要

预训练决定了模型能力的上限。微调只能"挖掘"预训练阶段积累的潜力，无法无中生有——模型在预训练中没见过知识，微调也"教"不出来。

这也解释了为什么业界对预训练数据的质量如此挑剔：GPT-4、Claude、DeepSeek V3 等顶尖模型的差距，很大程度上来自预训练数据工程的能力差异。

## 自监督学习：不需要标签的"自学"

### 下一 Token 预测

预训练的核心任务是**下一 Token 预测（Next Token Prediction）**——给定前文，预测下一个最可能出现的 token。

```
输入："今天天气"
模型预测："很"（概率最高）
```

这看起来简单，但要让模型预测得准，它必须理解：
- **语法**：主谓宾搭配
- **语义**：词与词之间的逻辑关系
- **世界知识**："天气"后面通常跟"好/坏/不错"
- **常识推理**：如果前文提到"下雨"，后面更可能是"大"

### 为什么不用标注数据

如果每个训练样本都需要人工标注"下一个词是什么"，成本将高到不可承受——互联网上有万亿 token 的文本，根本无法人工标注。

自监督学习的精妙之处在于：**训练数据本身就是答案**。"今天天气很好"这句话既是输入也是标签，模型从中自动提取语言规律。

### 掩码语言模型（MLM）

BERT 使用的是另一种自监督任务：掩码语言模型（Masked Language Model）。

```
原文："今天天气很好"
掩码："今天天气[MASK]好"
任务：预测 [MASK] 位置应该填什么
```

BERT 双向看到上下文，更适合理解任务（如文本分类），但 GPT 系列的"单向下一个 token 预测"在生成任务上更强，这也是两者路线分歧的根源。

## 预训练数据：从原始网页到高质量语料

### 数据来源分布

主流大模型的预训练语料构成：

| 数据来源 | GPT-4 (估计) | DeepSeek V3 |
|---------|------------|-------------|
| 网页文本（WebText等） | ~60% | ~45% |
| 书籍（Books） | ~20% | ~5% |
| 学术论文（ArXiv） | ~5% | ~4% |
| 代码（GitHub等） | ~5% | ~25% |
| 维基百科 | ~5% | ~3% |
| 其他 | ~5% | ~18% |

### FineWeb：高质量预训练数据的标杆

2024 年 Meta 发布的 **FineWeb** 数据集，是预训练数据工程的一个里程碑。它从 Common Crawl 的 15T token 原始网页中，通过系统化的质量过滤，最终得到 15B token 的 FineWeb-Edu 子集。

FineWeb 的质量过滤策略值得借鉴：

```python
# FineWeb 数据清洗核心逻辑（简化）
def quality_filter(text: str) -> bool:
    # 1. 语言检测：过滤非目标语言
    if not is_english(text):
        return False
    
    # 2. 重复检测：移除重复段落
    if detect_excessive_repetition(text):
        return False
    
    # 3. 教育价值评分（基于 LLama3 的分类器）
    edu_score = edu_classifier.score(text)
    if edu_score < 0.5:  # 只保留教育价值较高的内容
        return False
    
    # 4. 质量启发式规则
    if len(text) < 100 or len(text) > 50000:
        return False
    
    return True
```

关键洞察：**质量 > 数量**。DeepSeek V3 之所以在数学和代码任务上表现突出，很大程度上得益于其对代码数据集（DeepSeek-Coder）的精心构建。

### 数据处理的 Pipeline

```
原始数据 → 去重 → 语言过滤 → 质量过滤 → 安全过滤 → 分词 → 训练
```

每个环节都有大量工程细节：
- **去重**：URL 去重、文档级去重、段落级去重，防止模型记忆重复内容
- **质量过滤**：基于规则的过滤 + 基于模型的分类器（教育价值、可读性）
- **安全过滤**：去除有害内容、版权材料
- **分词**：BPE/BBPE 分词，中文场景用多语言分词器

## 预训练三阶段

### 1. 词汇扩展（Vocabulary Expansion）

如果预训练语料包含多语言，通常需要先扩展词表：

```python
# 假设原模型词表只有英文，增加中文词条
old_vocab_size = 50257
new_tokens = ["中", "文", "词", "表", "扩", "展", "..."]  # 新增约20000个中文token
new_vocab_size = old_vocab_size + len(new_tokens)
# 新增 token 的嵌入和输出层权重用小型中文模型初始化
```

### 2. 正常预训练（Standard Pre-training）

在高质量语料上训练，目标是语言建模损失最小化：

```python
# 简化的预训练循环
model = AutoModelForCausalLM.from_pretrained("base-model")

optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
scheduler = CosineAnnealingLR(optimizer, T_max=total_steps)

for step in range(total_steps):
    batch = next(data_loader)
    # Causal LM：每个位置预测下一个 token，损失仅计算 [next] 部分
    outputs = model(**batch)
    loss = outputs.loss
    
    loss.backward()
    optimizer.step()
    scheduler.step()
    optimizer.zero_grad()
```

### 3. 退火（Annealing / Final Training）

在训练末期，使用最高质量的数据（通常是精选的教育类、推理类数据），配合余弦退火学习率和更高的学习率，进行最后的"冲刺"训练。这一步对模型质量提升显著。

### Scaling Laws 的指导意义

OpenAI 2020 年发表的 *Scaling Laws for Neural Language Models* 给出了核心洞察：

> **模型的性能（困惑度）主要取决于三个因素：模型参数量 N、数据集大小 D、算力 C，三者服从幂律关系。**

这意味着：
- **增加模型规模**（更多参数）和**增加训练数据量**，都能提升模型性能
- 但存在一个**计算最优**点：当 D 和 N 的比例失调时，一边继续增大而另一方不变，收益会递减
- **Chinchilla 定律**进一步指出：对于给定的算力预算，最优策略是让模型参数量和训练 token 数**同比增长**

DeepSeek V3 的成功正是这一理论的实践：参数量 236B，训练 token 数高达 14.8T，远超同规模模型的数据量。

## 预训练的未来趋势

### 1. 数据质量越来越重要

从"堆数据"到"堆高质量数据"。LLM360-AMBER 和 FineWeb 的实验都证明：高质量小数据集可以打败低质量大数据集。

### 2. 合成数据崛起

GPT-4、Claude 都在训练中大量使用模型合成的数据。关键挑战是避免**模型崩溃（Model Collapse）**——用合成数据训练多代后，模型性能会退化。

### 3. 多模态预训练统一

文本、图像、代码、音视频的统一预训练（如 GPT-4o、Gemini）是下一个前沿。不同模态的"统一token"化是核心技术路线。

## 总结

| 维度 | 关键点 |
|------|--------|
| 核心任务 | 下一 Token 预测（自监督学习） |
| 数据来源 | 网页、书籍、代码、论文 |
| 质量工程 | 去重、过滤、教育价值评分 |
| 训练阶段 | 词汇扩展 → 正常预训练 → 退火 |
| Scaling Laws | N、D、C 服从幂律增长规律 |
| 成本 | 百亿参数模型训练成本数百万至千万美元 |

预训练是大模型能力的根基。理解预训练的数据工程和 Scaling Laws，是每个 AI 开发者进阶的必由之路。
