# 扩散语言模型：打破"自回归"的新范式

如果你跟了大模型领域足够久，可能已经对"自回归"这个词烂熟于心——GPT 就是自回归模型，一个 Token 接一个 Token，从左到右依次生成，就像写字一样一笔一划往外蹦。

但你有没有想过：**为什么大模型必须一个字一个字写？**

人类说话有时会脱口而出整句话，画画时不会从左到右一笔一笔画完才看整体，而是先有模糊的构思，再逐步细化。这种"先有大框架，再补细节"的方式，其实就是**扩散模型**的思路。

扩散模型在图像领域已经很成功了——DALL-E、Stable Diffusion 都是扩散模型。但 2024 年之后，一个新方向开始引起关注：**把扩散的思想搬到语言模型上**。这就是扩散语言模型——一种试图打破"自回归"统治的新范式。

今天我们来聊聊：扩散语言模型是什么？它跟自回归有什么本质区别？为什么它可能改变大模型的生成方式？

## 先复习：自回归的"缺陷"

自回归生成就是：给定前面所有 Token，预测下一个 Token。`P(x_t | x_1, ..., x_{t-1})`——条件概率，一步一步往右推。

这种方式有几个天然限制：

**1. 方向锁定**：只能从左到右。没法先写结尾再写开头，没法"先想好结局再铺垫过程"。就像写作文必须从第一句话开始——你改不了顺序。

**2. 全局一致性差**：每一步只看前面已经生成的部分，对"后面要说什么"没有预判。写到结尾时才发现前面铺垫错了，但已经来不及了——这就是为什么大模型经常"前后矛盾"。

**3. 速度瓶颈**：生成 N 个 Token 要跑 N 次推理。KV Cache 可以省掉重复计算，但步数不能减少。长文本生成慢，这是自回归的基因决定的。

打个比方：自回归就像**单行道**——只能往前走，不能回头，也不能绕路。而扩散模型想修一条**可以来回修改的高速公路**。

## 扩散模型的核心逻辑：从"噪声"到"清晰"

扩散模型在图像领域的工作方式，可以用一个简单的类比理解：

想象你有一张清晰的照片。你往上面不断撒"噪声"（就像往画面上泼墨水），泼了一百次之后，照片变成了一团完全看不清的灰色噪声。

**扩散模型做的事就是"反向泼墨水"**：给你一团纯噪声，它一步一步地把噪声去掉，还原出一张清晰的照片。

这个"从噪声到清晰"的过程，叫**去噪**（Denoising）。每一步去噪都是"把画面往清晰的方向推一点"——不是一步到位画好图，而是从模糊逐步细化。

具体来说：

1. **前向过程**（加噪）：清晰图片 → 加噪声 → 更模糊 → 再加噪声 → 纯噪声
2. **反向过程**（去噪）：纯噪声 → 去一点噪声 → 稍清晰 → 再去噪声 → 清晰图片

模型学的是反向过程——怎么从噪声一步步还原出有意义的内容。

## 扩散搬到语言上：从"噪声词"到"有意义的句子"

图像是连续数据（像素值可以任意取值），加噪就是加减像素值。但语言是离散数据——Token 是具体的词，你没法给"猫"这个 Token 加 0.1 的噪声变成"猫.1"。

这就是扩散语言模型的核心挑战：**怎么给离散的 Token 加噪和去噪？**

目前有几种方案：

### 方案一：嵌入空间扩散

Token 被映射成一个连续向量（嵌入向量）。既然向量是连续的，就可以在嵌入空间里做扩散——给向量加噪声、再去噪声，最后把还原的向量映射回最近的 Token。

打个比方：就像文字的"声音版"——你把每个词录成一段声波（嵌入向量），在声波上加噪去噪（连续操作），最后把还原的声波"翻译"回最近的词。

代表工作：Diffusion-LM（2022，Li et al.）。这是最早把扩散用在语言上的尝试。效果还不错，但有一个痛点：向量去噪后不一定正好落在某个 Token 的嵌入点上——需要找最近的 Token，这个过程会引入误差。

### 方案二：离散掩码扩散（MDLM）

更直觉的思路：语言里的"噪声"是什么？就是**把词换成随机词或者空位**。

想象一句话："猫坐在沙发上"。加噪的过程就是：

- 第一步：把"猫"换成随机词 → "狗坐在沙发上"
- 第二步：再把"坐"换掉 → "狗跑在沙发上"
- 第三步：把更多词换掉 → "鱼飞在天空中"

去噪就是反向操作：把乱词一步步替换回正确的词。这就像**填空题**——给你一个有很多空位的句子，模型一个一个把空填上，但可以不按顺序填。

代表工作：MDLM（Masked Diffusion Language Model，2024）。这个方案更贴近语言的本质——噪声就是"掩码"或"随机替换"，去噪就是"恢复正确词"。而且它打破了自回归的顺序限制——可以先填最确定的词，再填不太确定的词，顺序完全灵活。

### 方案三：对比自回归（Seduction）

还有一种巧妙的设计：不用纯扩散，而是把扩散和自回归**结合**。

核心思路：自回归生成太慢、太刚性，但它的局部预测能力很强。扩散模型灵活但离散处理有难度。能不能**用自回归的局部能力做扩散的每一步**？

具体来说：扩散模型的每一步去噪，不是"从噪声里恢复整句话"，而是"用自回归模型预测几个关键位置"，然后把预测结果当作去噪的一步。这样每一步都在利用自回归的强项，但整体流程是扩散式的——可以来回修改，不锁定方向。

你可以理解为：**自回归是"手"，扩散是"脑子"**。手负责精确操作（一步内预测几个词），脑子负责整体规划（决定先修哪里后修哪里）。

代表工作：Seduction（2025）、AR-Diffusion。这个方向还比较新，但思路很有吸引力——保留了自回归的成熟技术，又引入了扩散的灵活性。

## 扩散语言模型的优势在哪？

相比自回归，扩散语言模型有几个潜在优势：

### 1. 非顺序生成

自回归只能从左到右。扩散模型可以先填最确定的词，再填不确定的词——甚至可以先写结尾再写开头。

打个比方：自回归像手写一封信，必须从第一行写到最后一行。扩散像打字——你可以先敲标题，再回过头写开头，中间跳着写，最后拼成一封完整的信。

这对生成质量有什么影响？**全局一致性更好**。模型可以先确定关键信息（比如文章的核心论点），再围绕它填充细节，而不是一边写一边猜后面要说什么。

### 2. 可控修改

扩散模型天然支持"局部修改"——你可以指定某几个位置去噪，其他位置保持不变。就像编辑文档：选中几句话修改，其余不动。

自回归模型做不到这点——要改结尾就得重新生成整段，前面的内容不能保留。

### 3. 并行生成潜力

自回归每步只生成一个 Token。扩散模型每步可以同时去噪多个位置——如果步数能大幅减少，生成速度可能比自回归快很多。

想象一个流水线：自回归是一条单行道，车辆依次通过；扩散是多车道并行，几辆车同时在不同位置移动。

不过，目前扩散语言模型的步数通常还需要 10-50 步（每步处理整句），而自回归每步只生成 1 个 Token 但需要 N 步。哪个更快取决于文本长度和扩散步数的对比——短文本扩散可能更快，长文本目前自回归更稳。

## 为什么扩散语言模型还没取代自回归？

听起来扩散语言模型很有前景，但现实中 GPT、Claude、DeepSeek 全都用自回归。为什么？

**1. 离散数据天生不适合连续扩散**

图像是连续的——像素值可以平滑变化，加噪去噪很自然。语言是离散的——Token 是一个个具体的词，中间没有"过渡态"。你没法让模型从"猫"平滑过渡到"狗"，中间必须经过具体的词。这个离散性让扩散在语言上的效果天然比图像上差。

**2. Token 映射损耗**

在嵌入空间做扩散，最后要"投射回最近的 Token"。这个投射不是精确的——向量空间里的"最近点"不一定是你想要的词。就像翻译：意思对了但措辞不对。这个误差在长文本中会累积。

**3. 训练更复杂**

自回归的训练目标很清晰：预测下一个 Token。扩散的训练要学"在任意噪声水平下恢复正确 Token"——这比单步预测复杂得多，需要更精心的噪声调度和训练策略。

**4. 生态差距**

自回归经过 GPT-1 到 GPT-4 的多年迭代，整个技术栈（KV Cache、投机解码、量化加速）都是为自回归设计的。扩散语言模型还缺少对应的推理加速基础设施。

打个比方：自回归就像一条修了 30 年的高速公路，路平坦、加油站多、导航成熟。扩散语言模型是一条新修的路，风景可能更好，但路况差、配套少——大多数人还是会走老路。

## 前沿动态：谁在推进扩散语言模型？

虽然还没取代自回归，但扩散语言模型的研究正在加速：

- **MDLM**（2024）：离散掩码扩散，证明了纯离散扩散在语言上可行，生成质量接近自回归
- **Seduction**（2025）：自回归+扩散的混合方案，用自回归做扩散的每一步，兼具两者优势
- **LLM-Diffusion**：微软等团队在探索把大模型的预训练能力嫁接到扩散框架上
- **多 Token 并行生成**：本质上跟扩散的"一步多位置去噪"思路相通——这也是 MTP（多令牌预测）的方向

一个有趣的观察：**投机解码和 MTP 其实都在"打破自回归的逐 Token 生成"**。虽然它们不是扩散模型，但核心动机跟扩散是一样的——"能不能更快、更灵活地生成？"

所以扩散语言模型不是孤立的探索，它跟投机解码、MTP、并行生成是同一个大方向的不同路线：**让大模型不再被自回归的顺序锁死**。

## 给前端开发者的启示

你不会去训练扩散语言模型，但理解它有几个好处：

1. **理解大模型的局限**：自回归的"逐字生成"不是最优方案，只是目前最成熟的。未来可能会有更灵活的生成方式——这会影响你用的 API 的设计。
2. **理解可控生成**：扩散模型的"局部修改"能力，可能意味着未来的大模型 API 会支持"指定修改位置"而不需要重写整段——这对代码编辑、文档修改场景很有价值。
3. **关注并行生成**：如果扩散语言模型成熟，生成速度可能大幅提升——这对实时交互场景（聊天、代码助手）意义重大。

现在扩散语言模型还在早期，但方向是明确的：**打破自回归的单行道，让生成更灵活、更可控、更快**。就像图像领域扩散取代了 GAN，语言领域也许会有类似的范式切换——只不过时间可能更长，因为语言的离散性是个硬骨头。

---

> 扩散语言模型的核心思想很简单：不是"一个字一个字往外蹦"，而是"先有一团模糊，再逐步清晰化"。这个思路在图像领域已经成功了，搬到语言上还有离散性的硬骨头要啃。但无论最终谁赢，"打破自回归顺序锁"这个方向是确定的——未来的大模型，很可能不再是一条单行道。
