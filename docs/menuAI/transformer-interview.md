# 2025年AI大模型高频面试题之 Transformer（含详细答案解析）

> 这篇文章梳理了 15 道 Transformer 高频面试题，附上详细答案。不是死记硬背，是把每个知识点从"会背"变成"真懂"。适合准备 AI 算法面试的前端开发者查漏补缺。

---

## 一、先画个整体认知：Transformer 有哪几块？

在说面试题之前，先把 Transformer 的骨架理清楚。整体分三块：

```
┌────────────────┐     ┌────────────────┐
│    Encoder      │────▶│    Decoder     │
│  (编码器堆叠×6) │     │  (解码器堆叠×6) │
└────────────────┘     └────────────────┘
        │                     ▲
        │              ┌───────┴───────┐
        │              │  交叉注意力层   │
        │              │ (Cross-Attention)│
        │              └───────────────┘
```

**Encoder 核心组成（每层）：**
- Multi-Head Self-Attention（多头自注意力）
- Feed-Forward Network（全连接前馈网络）
- 每个子层都有 **残差连接 + Layer Norm**

**Decoder 核心组成（每层）：**
- Masked Multi-Head Self-Attention（带掩码的自注意力）
- Cross-Attention（交叉注意力，接 Encoder 输出）
- Feed-Forward Network

> 记住这个结构，很多面试题都是从这里"挖"出来的。

## 二、Self-Attention 计算公式（必背）

面试官最爱让你"手推"这个公式：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

**三个步骤拆解：**

```
Step 1: QK^T     → 计算"每个词对每个词"的相似度得分
Step 2: ÷ √d_k   → 缩放，防止点积过大导致梯度消失
Step 3: softmax  → 归一化成概率分布（注意力权重）
Step 4: × V      → 加权求和，得到最终输出
```

> 如果面试官追问："为什么偏偏用 `√d_k` 缩放？"
> **答：** 当 `d_k` 较大时，`QK^T` 的点积值会很大，softmax 会被推入饱和区（梯度接近 0），导致训练困难。`√d_k` 相当于把方差拉回 1，保证梯度稳定。

## 三、Transformer 优缺点（理解大于背诵）

### 优点

| 优点 | 说明 |
|------|------|
| **并行能力强** | 不像 RNN 必须"一步步"处理，Transformer 可以一次性看完整个序列，训练效率大幅提升 |
| **长距离依赖** | 任意两个词之间"一步直达"，不再有 RNN 的梯度消失问题 |
| **灵活性和泛化能力** | 一个架构通吃翻译、生成、分类、语音 |
| **可扩展性强** | 加层数、加维度、加头数，都能继续涨点 |
| **性能领先** | BLEU、GLUE 等基准上刷新了历史记录 |

### 缺点

| 缺点 | 说明 |
|------|------|
| **计算资源密集** | 自注意力的计算量是 **O(n²)**，长序列直接爆显存 |
| **可解释性差** | 注意力权重只是一个参考，模型内部决策过程仍是黑箱 |
| **过拟合风险** | 参数量巨大，少量数据容易"背答案" |
| **训练调参复杂** | 学习率、层数、头数、维度……最优组合需要大量实验 |
| **局部信息捕获弱** | 全局 attention 反而对"相邻词"关系不如 CNN/RNN 敏感 |
| **长序列挑战** | 非常长的文本（> 32K tokens）性能会明显下降 |

> 面试加分项：主动说"针对这些缺点业界有哪些优化方案"（如 Sparse Attention、FlashAttention、Ring Attention 等）。

## 四、Encoder 和 Decoder 怎么"对话"？

Encoder 编码输入，Decoder 生成输出，两者通过 **Cross-Attention（交叉注意力）** 连接。

**交互流程：**

```
1. Encoder 处理输入序列，输出每个词的上下文表示（K 和 V）

2. Decoder 在生成第 t 个词时：
   - Query：来自 Decoder 自身（当前要生成的词）
   - Key & Value：来自 Encoder 的输出（整个输入序列的上下文）
   
3. Cross-Attention 计算：
   Attention(Q_decoder, K_encoder, V_encoder)
   → Decoder 在生成时"参考"了整个输入序列的信息
```

**一个比喻：**
> 翻译任务是 Encoder-Decoder 交互的经典场景。Encoder 像是一个"阅读理解者"，把原文读完后标注好重点；Decoder 在翻译时，随时回头查这个"阅读理解者"的笔记，决定下一个词怎么写。

## 五、Q/K/V 为什么要做线性变换？

**核心原因：增加表达能力的"泛化半径"。**

没有线性变换时，Q、K、V 就是原始输入的直接投影，学到的关系是"固定的"。
加了 `W_q`、`W_k`、`W_v` 三个参数矩阵后：

- Q 和 K 被投影到**不同的子空间**，可以从不同角度衡量相似度
- 这和 SVM 用核函数把向量映射到高维空间是同样思想——**用非线性变换捕获更复杂的模式**

## 六、为什么注意力矩阵用乘法而不是加法？

**答案：计算速度 + 硬件友好。**

| 操作 | 特点 |
|------|------|
| 矩阵乘法 | 有高度成熟的 GPU/TPU 加速实现（cuBLAS 等），利用率高 |
| 加法 + 激活 | 虽然数学上等价，但无法利用并行硬件加速 |

实践中，`QK^T` 的矩阵乘法可以一次并行算出所有词的注意力分数，速度远快于逐元素加法。

## 七、Padding Mask 怎么做？

**目的：** 输入序列长度不一，需要 padding 补齐，但模型不该关注这些"补进来的无意义词"。

**操作：**
```python
# 在 QK^T 之后、softmax 之前
attention_scores = attention_scores.masked_fill(mask == 0, -1e9)
# padding 位置的分数设为极小值，softmax 后概率接近 0
```
等价于：把这些位置乘以一个"忽略权重"，让模型根本"不看"它们。

## 八、残差连接的意义（重点理解）

### 结构公式

```
Output = LayerNorm(x + Sublayer(x))
```

### 五个核心意义

| 意义 | 解释 |
|------|------|
| **缓解梯度消失** | 梯度可以绕过子层直接传回输入层 |
| **加速收敛** | 底层梯度更稳定，学习更高效 |
| **促进深层训练** | 残差通道让信息在深层网络中"抄近道" |
| **保留原始信息** | 即使堆了 12 层，底层特征也不会被覆盖 |
| **支持特征重用** | 低层特征可直接被高层引用，不需要重新学习 |

> **面试高频追问：**"残差连接为什么有效？"
> **标准答案：** 因为它提供了"恒等映射"这条通道。网络学习的目标从"直接拟合底层到顶层的复杂函数"，变成了"拟合残差"。当残差接近 0 时，深层网络至少能退化成浅层网络，性能不会退化。

## 九、为什么用 Layer Norm 而不是 Batch Norm？

**一句话答案：** Batch Norm 依赖 batch 内统计量，NLP 任务 batch 大小往往很小且序列长度不一致，LN 更稳定。

| 对比 | Layer Norm | Batch Norm |
|------|-----------|-----------|
| 归一化方向 | 对单个样本的所有特征 | 对 batch 内所有样本的同一特征 |
| 序列任务 | ✅ 稳定 | ❌ 不稳定（序列长度不一致时问题大） |
| RNN / Transformer | ✅ 常用 | ❌ 基本不用 |
| 训练/推理一致性 | ✅ 一样 | ❌ 不一样（batch 统计量在推理时要用移动平均） |

> 在 Transformer 中，每个样本的每个位置都有自己的归一化参数，不受 batch 影响，所以 LN 是标准配置。

## 十、Decoder 的自注意力和 Encoder 有什么区别？

**本质区别：Decoder 有 causal mask（因果掩码）。**

### Encoder 多头注意力
- **无掩码**，每个词可以"看"序列中的所有其他词
- 双向上下文

### Decoder 多头注意力（分两层）

**第一层：Masked Self-Attention**
- 加了 causal mask，生成第 `t` 个词时**只能看到前 t-1 个词**
- 保证**自回归特性**：模型在预测当前词时，不会"偷看"答案
- 这是 **并行训练** 的关键——用 mask 模拟串行依赖

**第二层：Cross-Attention（与 Encoder 交互）**
- Query 来自 Decoder 自身
- Key 和 Value 来自 Encoder 最终输出

## 十一、Decoder 能并行化吗？

**训练时：可以并行。** 靠的就是 causal mask，把"不能看的词"屏蔽掉，整个序列可以一起算。

**推理时：不行，必须逐词生成。** 因为第 `t` 个词的输入依赖第 `t-1` 个词的输出，这是序列依赖，无法并行。

> **面试加分项：** 解释清楚"训练并行 vs 推理不并行"的根本原因。

---

## 十二、计算量最大的部分是哪里？

**答案：多头自注意力机制。**

复杂度分析：

| 部分 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| Self-Attention | **O(n² · d)** | O(n²) 存储注意力矩阵 |
| Feed-Forward | O(n · d²) | O(n · d²) |
| 多头 | × h（头数） | × h |

- `n` = 序列长度
- `d` = 模型维度
- `h` = 头数

> `n²` 是瓶颈！当序列长度翻倍，计算量涨 4 倍。这就是为什么长上下文是工程难题。

## 十三、Transformer vs LSTM vs 纯 FFN，提升在哪里？

| 模型 | 能力 |
|------|------|
| 纯 FFN | 无顺序感知，词序信息完全丢失 |
| LSTM | 理解语序，解决梯度消失/爆炸 | 
| **Transformer** | 一步到位解决长距离依赖 + 并行训练 |

**核心进化链：**
```
FFN（无序）→ LSTM（理解顺序，解决短距离依赖）
→ Transformer（全序列可见，解决长距离依赖，可并行）
```

## 十四、面试最容易被问到的 5 道追问

### Q1：为什么 Multi-Head 不用一个头？
**A：** 多个头能让模型在不同的子空间学习不同的注意力模式，有的头关注语法关系，有的头关注语义相似度……矩阵总维度不变，但表达能力更强。

### Q2：残差连接为什么加在每个子层而不是只在特定位置？
**A：** 每个子层都有信息损失和梯度风险，均匀加残差保证每一层都能直接获得原始信号，避免"哪一层塌了"都不知道。

### Q3：Attention 计算时 Q、K、V 能不能用同一个矩阵？
**A：** 可以，但效果差。用同一个矩阵，Q 和 K 被投影到同一空间，学到的相似度模式单一；分开投影让模型从更多角度衡量相关性。

### Q4：Transformer 对超长文本怎么处理？
**A：** 常用方法：Sparse Attention（稀疏注意力）、FlashAttention（IO感知的精确注意力）、Ring Attention（分布式长上下文）、滑动窗口 + 全局注意力的分层策略。

### Q5：如果让你优化 Transformer 的推理速度，你会怎么做？
**A：** KV Cache 缓存已计算结果 + 避免重复计算、量化（INT8/INT4）减小内存带宽压力、投机解码（Speculative Decoding）小模型猜、大模型验证、剪枝不重要的注意力头。
