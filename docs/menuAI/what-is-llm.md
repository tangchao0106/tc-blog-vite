# 大模型是什么：从"猜词游戏"到"数字大脑"

## 一个简单的类比

想象你在玩一个游戏：给你一句话的前半段，你来猜下一个字。

> "今天天气真___"

你会猜"好"、"热"、"冷"、"不错"。如果我再告诉你："今天是晴天，气温 28 度"，你会更自信地猜"好"或"热"。

**大语言模型（LLM）本质上就是一个超级加强版的"猜词游戏"。** 它学习了海量文本中字词之间的统计规律，当你给它一段文字，它会计算每个可能的下一个词的概率，然后选一个"最合理"的词接上。

## 三个核心概念

### 1. Token：模型的"最小思考单元"

模型不直接处理"字"，它处理的是 **Token**——介于"字"和"词"之间的语义单元。

```text
输入文本："人工智能正在改变世界"
         ↓ Tokenizer 分词
Token序列：["人工智能", "正在", "改变", "世界"]
         ↓ 映射到词表 ID
数字序列：[38472, 1045, 2893, 1582]
```

每个数字 ID 对应词表（Vocabulary）中的一个 token。GPT-4 的词表约 10 万个 token，DeepSeek 的词表约 12.8 万个。模型始终在和这些数字打交道，而不是汉字或英文单词。

### 2. 神经网络：一个巨大的概率计算器

如果你把词表里的 10 万个 token 排成一排，模型做的事情就是：

> **给定前文，计算出 10 万个 token 各自出现在下一个位置的概率。**

```text
输入："中国的首都是"
      ↓ 神经网络计算
输出概率：
  北京: 0.92  ← 最高
  上海: 0.03
  广州: 0.01
  ...（剩余 99997 个 token 的概率）
```

这个过程背后是数十亿甚至数千亿个参数（weights）在做矩阵乘法。**参数就是模型从训练数据中学到的"知识"，本质上是一串浮点数字。**

```python
# 一个极度简化的"模型"思维方式
# 真实的 LLM 比这复杂数百万倍

def tiny_model(context: str) -> dict:
    """根据上下文预测下一个词的概率分布"""
    patterns = {
        "中国的首都是": {"北京": 0.92, "上海": 0.03, "广州": 0.01},
        "1+1=": {"2": 0.99, "3": 0.005, "1": 0.005},
    }
    return patterns.get(context, {})
```

### 3. 参数：模型记忆的"脑细胞"

| 模型 | 参数量 | 类比 |
|------|:------:|------|
| GPT-2 | 1.5 亿 | 老鼠大脑 |
| GPT-3 | 1750 亿 | 猫的大脑 |
| GPT-4 | ~1.8 万亿（MoE） | 接近人脑复杂度 |
| DeepSeek V3 | 6710 亿（MoE，激活 370 亿） | 专家团队协同 |

> **参数越多 ≠ 一定越聪明**，但更多参数通常能捕捉更复杂的语言规律。

## 训练 vs 推理：两段截然不同的人生

这是理解大模型最重要的概念区分之一。

### 训练（Training）：模型的"求学期"

```
                    ┌──────────────┐
海量文本 ──────────→│   大模型训练   │──→ 模型权重文件
（万亿级 token）    │ （数周～数月）  │   （几百 GB）
                    └──────────────┘
```

训练过程的核心是**自监督学习**：

1. 拿来一段文本，遮住后半部分
2. 让模型预测被遮住的部分
3. 对比预测结果和真实文本，"批评"模型
4. 调整参数，降低预测误差
5. 重复万亿次

```text
训练数据："床前明月光，疑是地上霜。举头望明月，低头思故乡。"

步骤1：喂入 "床前明月光，疑是地上霜。举头望明月，"
步骤2：模型预测 → "故乡。"
步骤3：真实答案是 "低头思故乡。" → 错了，扣分！
步骤4：梯度下降，调整参数，下次更容易预测对
```

**训练一次 GPT-4 级别的模型，成本高达数千万到上亿美元**，需要上万张 GPU 跑几个月。

### 推理（Inference）：模型"上班干活"

```
                    ┌──────────────┐
你的 prompt ───────→│  已训练的模型  │──→ 逐 token 输出
（几百 token）      │ （前向传播）   │   （流式生成）
                    └──────────────┘
```

推理是**纯前向计算**，不再修改参数：

1. 接收输入 prompt
2. 逐层计算（矩阵乘法）
3. 输出下一个 token 的概率分布
4. 选一个 token 作为输出
5. 把新 token 接在末尾，重复步骤 2～4

**推理是"只读"操作，不会改变模型的任何参数。** 这就是为什么 ChatGPT 不会"记住"你和它聊过什么——对话历史只存在于当前会话的上下文中，不写入模型权重。

### 训练 vs 推理 对比

| 维度 | 训练 | 推理 |
|------|------|------|
| **目的** | 学习知识 | 回答问题 |
| **参数** | 不断更新 | 冻结不变 |
| **硬件需求** | 数千张 GPU | 1～8 张 GPU |
| **耗时** | 数周至数月 | 毫秒到秒级 |
| **成本** | 千万美元级 | 几分钱/次 |
| **数据方向** | 反向传播（反向） | 前向传播（正向） |
| **类比** | 学生刷题备考 | 学生上考场答题 |

## 大模型的"大"在哪？

### 1. 数据量大

GPT-4 的训练数据约 **13 万亿 token**——相当于把人类历史上几乎所有公开书籍、论文、网页、代码都读了一遍。

### 2. 参数多

从 GPT-2 的 15 亿到 GPT-4 的 1.8 万亿，增长了 1000 多倍。每增加一个数量级的参数，模型就会涌现出新的能力（这种现象被称为"涌现" Emergence）。

### 3. 算力大

训练 GPT-4 级别的模型需要约 **2 万张 A100 GPU 跑 90～100 天**，总耗电量相当于一个小型城镇一年的用电量。

## 大模型为什么"会犯错"

理解了大模型的本质是"猜词游戏"，就能理解它的局限：

1. **不是真正"理解"**：模型只是学到了字词之间的统计相关性，没有真正的意识和理解
2. **概率驱动**：它给出的不是"正确答案"，而是"最可能的接续"
3. **幻觉（Hallucination）**：当训练数据中没有相关信息，模型会"编造"——因为从概率上，总有一个词比沉默更合理
4. **上下文窗口限制**：模型不能无限回顾历史，长对话会"遗忘"早期内容

## 你应该记住什么

> 大语言模型 = 海量参数 + 海量训练数据 + 超级算力
>
> 它做的事 = 给定上文，猜下一个 token 的概率分布
>
> 训练 = 调整参数让猜得更准，推理 = 用固定参数干活
>
> 它不是真正的"智能"，但统计规律足够强大时，看起来和智能很难区分。
